# 03. サイト設計

## 架空企業: Nordic Works

| 項目 | 設定 |
|------|------|
| 社名 | Nordic Works |
| 業種 | B2B SaaS |
| 事業領域 | 北欧式の働き方・組織設計を支援 |
| 主要サービス | 心理的安全性スコア計測ツール、リモートワーク文化診断、組織デザインコンサル |
| メディアテーマ | リモートワーク、心理的安全性、組織デザイン、北欧の働き方文化 |
| ターゲット | 30-40代の経営層・人事担当者・組織開発担当 |
| ブランドトーン | 知的・洗練・北欧的ミニマル |

## コンセプトの根拠

このコンセプトを選んだ理由は3つ:

1. **実案件に近い構造**: B2B SaaSの「サービス紹介 + ナレッジ発信」型は実需が多く、採用面接で「あ、こういう仕事できるんだ」と即理解される
2. **コーポレートとメディアの両立**: 実案件で最も多い構造と一致
3. **デザイン軸が明確**: 北欧×ミニマルという具体的方向性で、AI生成にありがちな「無個性なシャレた感じ」を回避できる

## デザイン方針

- ブランドカラー: 北欧的なニュアンスカラー（オフホワイト、ダークネイビー、サブカラーにマスタード or テラコッタ）
- 余白を大きく取るエディトリアルなレイアウト
- 写真は自然光・落ち着いた色温度のもの（Unsplash等から選定）
- アニメーションは控えめ、丁寧、洗練を意識
- フォント: 見出しに北欧的なディスプレイ書体（Söhne, Inter Tight, GTAmerica等）、本文に読みやすいSerif/Sans-serif

## サイト構造

```
Nordic Works
├ コーポレート
│  ├ トップページ（最新記事・サービス紹介・CTA）
│  ├ サービス紹介（一覧 + 詳細 ×3〜4種）
│  ├ 会社概要（About）
│  ├ 採用情報（一覧 + 詳細）
│  └ お問い合わせ
│
├ メディア（"Insights" 等の名称想定）
│  ├ 記事一覧（最新・人気・特集）
│  ├ 記事詳細
│  ├ カテゴリー別一覧（業界別・トピック別）
│  ├ タグ別一覧
│  ├ 著者プロフィール一覧
│  ├ 著者別記事一覧
│  ├ 特集（厳選コンテンツのキュレーション）
│  └ 検索ページ
│
└ 共通
   ├ ヘッダー（ナビゲーション）
   ├ フッター
   ├ 404ページ
   └ プレビューモード
```

## ページ仕様

### トップページ

- ヒーローセクション（キャッチコピー + サブコピー + CTA）
- 最新記事一覧（3〜6件）
- 注目の特集（1〜2件）
- サービス紹介（カード3〜4枚）
- 採用情報への導線
- お問い合わせCTA

### 記事詳細ページ

- カバー画像（アイキャッチ）
- タイトル・サブタイトル
- 著者情報・公開日・更新日・読了時間
- 自動目次（フローティング表示）
- 本文（マークダウン or Gutenberg JSON）
- スクロール進捗バー
- 関連記事レコメンド
- 著者プロフィール（記事末尾）
- 他記事へのナビゲーション

### サービス詳細ページ

- ヒーロー画像 + キャッチコピー
- サービス概要
- 機能リスト
- 料金表
- FAQ
- 導入事例リンク
- お問い合わせCTA

### 採用情報詳細ページ

- ポジション名
- 雇用形態・勤務地
- 仕事内容
- 必須スキル
- 歓迎スキル
- 待遇
- 応募フォーム or 応募ボタン

### 著者プロフィールページ

- 顔写真
- 肩書き
- 自己紹介
- SNSリンク
- 執筆記事一覧

### 検索ページ

- 検索ボックス
- フィルター（業界・トピック・著者）
- 結果一覧（ハイライト付き）
- ページネーション

## URL設計

| ページ | パス |
|--------|------|
| トップ | `/` |
| 記事一覧 | `/articles` |
| 記事詳細 | `/articles/[slug]` |
| カテゴリー別 | `/category/[slug]` |
| タグ別 | `/tag/[slug]` |
| 特集一覧 | `/features` |
| 特集詳細 | `/features/[slug]` |
| 著者一覧 | `/authors` |
| 著者別 | `/authors/[slug]` |
| サービス一覧 | `/services` |
| サービス詳細 | `/services/[slug]` |
| 採用一覧 | `/careers` |
| 採用詳細 | `/careers/[slug]` |
| 会社概要 | `/about` |
| お問い合わせ | `/contact` |
| 検索 | `/search` |

## レスポンシブ対応

- モバイル: 〜768px
- タブレット: 769px〜1024px
- デスクトップ: 1025px〜

モバイルファースト設計で、Tailwind CSSのbreakpoint（sm/md/lg/xl）を活用。

## アクセシビリティ要件

- セマンティックHTML（適切な見出しレベル、landmark要素）
- キーボード操作対応
- 適切なコントラスト比（WCAG AA以上）
- 画像のalt属性
- フォームのラベル関連付け
- フォーカス可視化
